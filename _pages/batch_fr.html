Fonctionnement

Ensemble de commandes de script permettant d'automatiser la création, modification et suppression de portfolios, ainsi que la création ou suppression d'utilisateurs.

À partir d'un ou plusieurs modèles de portfolios, d'un script batch et d'un fichier CSV, KARUTA crée les utilisateurs et leurs portfolios. Le fichier CSV contient les données relatives à chaque utilisateur à créer. Le script de batch indique les opérations à effectuer : créer l'utilisateur, créer son ou ses instances de portfolios à partir de modèles de portfolio, partager les nouveaux portfolios avec les utilisateurs dans des rôles spécifiés, remplir les données de l'utilisateur dans les portfolios, etc.

Exemple de script batch

Ce script correspond à l'algorithme suivant :

Créer le projet pour la cohorte
Pour chaque ligne du fichier CSV
 Créer le compte pour l'étudiant si il n'existe pas
 Créer le portfolio de l'étudiant
 Mettre à jour le champ Nom dans le portfolio
 Partager le portfolio avec l'étudiant dans le rôle "etudiant"

Exemple de fichier CSV

Le fichier CSV contient les informations nécessaires à l'exécution du batch.

Format du fichier CSV :

  • Ligne 1 Les noms des variables globales. Les variables globales sont les variables qui ont les mêmes valeurs pour chaque utilisateur à créer (par exemple, les codes des modèles de portfolio à instancier, un code de programme ou une session). Dans le script de batch, les références aux variables globales sont précédées de //.
    model_code en cellule A1 est un mot réservé : c'est le code du portfolio contenant le script de batch à exécuter.
  • Ligne 2 Les valeurs des variables globales. Dans l'exemple présenté la valeur de model_code est exemple.batch-creer-cohorte : c'est le code du portfolio contenant le script de batch.
  • Ligne 3 Les noms des variables locales. Les variables locales sont les variables qui ont des valeurs différentes pour chaque utilisateur à créer (par exemple, le nom de l'étudiant, son prénom, courriel, etc).
  • Lignes 4 et + Les valeurs des variables locales, un utilisateur par ligne. En général, les instructions du script de batch seront exécutées pour chacune des ces lignes.

NOTE IMPORTANTE SUR LA SYNTAXE DES NOMS DE VARIABLES
Les noms des variables globales et locales ne peuvent PAS inclure les caractères suivants : trait d'union (-), plus (+), astérisque (*), barre oblique (/), pourcentage (%) ou dièse (#).
Le caractère de soulignement (_) est permis.

Pour chaque ligne

L'élément Pour chaque ligne est une boucle et est généralement le point de départ d'un script de batch. Les commandes à l'intérieur de l'élément Pour chaque ligne seront exécutées pour chaque ligne du fichier CSV (lignes 4+).

Pour chaque arbre

L'élément Pour chaque arbre est une boucle. Les commandes à l'intérieur de la partie Actions seront exécutées pour chaque portfolio dont le code correspond au critère indiqué dans l'attribut Partie du code de portfolio.

Champs à renseigner :

  • Référence arbre Attribuer un identifiant qui sera utilisé pour faire référence à ce portfolio dans les commandes de la partie Actions.
  • Partie du code de portfolio Partie du code de portfolio unique aux portfolios désirés. Ajouter des éléments variable pour des variables globales ou locales du fichier CSV et/ou des éléments valeur pour spécifier directement des valeurs.
  • Actions Les commandes à exécuter pour chacun des portfolios sélectionnés par le critère Partie du code de portfolio.

Si Alors Sinon

Instruction classique Si (test) Alors [instructions] Sinon [instructions].

Champs à renseigner :

  • Action du Si Ajouter une composante de batch. Doit retourner vrai ou faux à la fin de l'action : succès ou échec.
  • Action du Alors Ajouter les composantes de batch à exécuter en cas de succès.
  • Action du Sinon Ajouter les composantes de batch à exécuter en cas d'échec.

Créer utilisateur

Créer un utilisateur. Ne pas oublier de remplacer les valeurs par défaut de cet élément par vos propres noms de variables locales (ligne 3 du fichier CSV).

Champs à renseigner :

  • Identifiant Identifiant unique du l'utilisateur qui servira de login dans le système Karuta (par exemple un matricule ou courriel).
  • Prénom Prénom de l'utilisateur.
  • Nom Nom de famille de l'utilisateur.
  • Courriel Courriel de l'utilisateur.
  • Mot de passe Le mot de passe de l'utilisateur dans le système Karuta.
  • Créateur? Le compte utilisateur créé doit-il avoir les droits d'un designer Karuta? (En général, non!)

Désactiver utilisateur

Désactiver un compte utilisateur. Désactiver un compte plutôt que de le supprimer permet de conserver les partages aux portfolios et les associations aux groupes d'utilisateurs en prévision d'une réactivation ultérieure.

Champ à renseigner :

  • Login utilisateur Login du compte utilisateur. Ajouter un élément variable pour puiser l'identifiant dans une variable globale ou locale du fichier CSV, ou un élément valeur pour spécifier directement l'identifiant de l'utilisateur.

Importer nœud

Importer un élément dans un portfolio, par exemple pour ajouter une page ou une composante réutilisable. L'élément à importer doit exister dans un autre portfolio. Une copie sera faite, comme lors de l'utilisation des menus aux utilisateurs.

Champs à renseigner :

  • Cible Section, page ou sous-section où importer l'élément désiré. Il y a trois options pour spécifier le nœud cible : sélection d'une référence d'arbre + tag sémantique du nœud, dernier nœud importé, ou par UUID. (La référence arbre doit préalablement être attribuée dans un élément Créer arbre ou Sélectionner arbre.)
  • Source Élément à importer dans le portfolio. Il y a deux options pour spécifier le nœud à copier : sélection d'une référence d'arbre + tag sémantique du nœud ou par UUID. (La référence arbre doit préalablement être attribuée dans un élément Créer arbre ou Sélectionner arbre.)
