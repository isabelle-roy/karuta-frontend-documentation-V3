La fonction Partage par l’utilisateur – Donner accès à une partie de portfolio

Contexte : Dans un portfolio Karuta, le designer doit spécifier les ressources, les rôles et les actions permises.  Par exemple :

·       Un étudiant peut éditer une ressource texte.

·       Un professeur peut donner son appréciation à l’aide d’une grille de correction et Montrer/Cacher le résultat.

·       Un étudiant peut éditer un document et ensuite le soumettre.

·       Un pair peut éditer un commentaire et ensuite le soumettre.

·       Tous les usagers (rôle all) qui ont accès au portfolio d’un étudiant peuvent voir une ressource.

·       Etc.

Lors de l’instanciation du portfolio (création de la copie du portfolio modèle pour un étudiant donné), les différents rôles sont associés à des usagers précis. Par exemple, l’étudiant = Louis, le professeur = André et le pair = Simon (et all = public si requis). On dit aussi que le portfolio est partagé avec des utilisateurs.

Partage par l’utilisateur : Le designer peut aussi prévoir que des portfolios complets, de parties de portfolios ou même des ressources spécifiques pourront être partagées par un usager donné à l’aide d’un bouton. Les usagers qui recevront le partage en seront avisés par courriel (le courriel contient un hyperlien vers le portfolio partagé). Ils pourront alors intervenir dans le portfolio selon le rôle qui leur sera assigné.  Le partage permet deux possibilités : 

1.     avertir un acteur du portfolio qu’il doit intervenir;

2.     permettre à un usager qui n’a pas actuellement les droits d’intervenir dans un portfolio de le faire.

 

Dans le portfolio :

 

Ainsi, si dans son portfolio l’étudiant clique sur Partager avec un pair, le système envoi un courriel à un pair avec un hyperlien vers une section du portfolio dans laquelle le pair pourra inscrire son commentaire.

Dans le courriel :

 

La spécification du partage se fait dans l’onglet métadonnées d’une section, d’une page, d’une sous-section ou même d’une ressource.

 

Dans les métadonnées :

 

Les conditions de ce partage sont très sophistiquées; voici quelques cas d’utilisation usuels :

 

Cas 1 – L’étudiant peut montrer son portfolio à n’importe qui (usager Karuta ou non) en mode lecture seulement (aucun droit d’édition).

Partager
rôle qui partage[,rôle destinataire,rôles et/ou courriels,niveau (0-3),durée de vie du lien (en heures),button label@en/libellé du bouton@fr[,condition]][;autre partage]

etudiant

 

 

En cliquant sur le bouton Partager (), l’étudiant n’aura qu’à inscrire l’@dresse de la personne souhaitée. Celle-ci recevra par courriel un hyperlien lui donnant accès, de façon permanente et en mode lecture seulement, à la partie du portfolio partagée.

Note 1 : Le designer n’a qu’à spécifier le rôle de l’usager qui va faire le partage; les paramètres indiqués entre [ ] ne sont pas requis.

Note 2 :  Lors de l’instanciation du portfolio, le rôle all doit être partagé avec l’utilisateur public.

 

Cas 2 – L’étudiant demande à son professeur de procéder à son évaluation; le professeur possède un compte Karuta et a été assigné au portfolio de cet étudiant dans le rôle professeur lors de l’instanciation.

Partager
rôle qui partage[,rôle destinataire,rôles et/ou courriels,niveau (0-3),durée de vie du lien (en heures),button label@en/libellé du bouton@fr[,condition]][;autre partage]

etudiant,professeur,professeur,1,12,Demander une évaluation@fr

 

Le professeur reçoit un courriel et n’a qu’à cliquer sur l’hyperlien pour procéder à l’évaluation.

 

Cas 3 – L’étudiant demande à son responsable de stage de compléter son évaluation; le responsable de stage n’a pas de compte Karuta.

Partager
rôle qui partage[,rôle destinataire,rôles et/ou courriels,niveau (0-3),durée de vie du lien (en heures),button label@en/libellé du bouton@fr[,condition]][;autre partage]

etudiant,evaluateur,yves.lacoste@airfrance.com,3,12,Demander une évaluation@fr

 

L’évaluateur externe reçoit un courriel et n’a qu’à cliquer sur l’hyperlien pour procéder à l’évaluation; il n’a pas à s’identifier dans le système Karuta. L’hyperlien est valide pendant 12 heures.

 

Cas 4 – L’étudiant demande à un de ses collègues d’inscrire un commentaire dans son portfolio (évaluation par les pairs); le collègue possède un compte Karuta.

Partager
rôle qui partage[,rôle destinataire,rôles et/ou courriels,niveau (0-3),durée de vie du lien (en heures),button label@en/libellé du bouton@fr[,condition]][;autre partage]

etudiant,pair,?,1,12,Demander un commentaire@fr

 

Le collègue reçoit un courriel et n’a qu’à cliquer sur l’hyperlien pour écrire son commentaire.

Note :  Le symbole « ? » fait en sorte que lorsque l’étudiant cliquera sur le bouton, une fenêtre s’ouvrira pour lui permettre de spécifier le courriel de son collègue. On fait ici l’hypothèse que l’étudiant a un grand nombre de collègues et que le designer n’a pas spécifié de façon précise les collègues qui seront mobilisés, de façon à donner plus de latitude à l’étudiant dans le choix du pair qui pourra l’évaluer.

 

 Cas 5 – L’étudiant peut montrer son portfolio à n’importe qui (usager Karuta ou non) en mode lecture seulement (aucun droit d’édition) mais le partage a une durée limitée de 24 heures.

Partager
rôle qui partage[,rôle destinataire,rôles et/ou courriels,niveau (0-3),durée de vie du lien (en heures),button label@en/libellé du bouton@fr[,condition]][;autre partage]

Etudiant,all,?,4,24,Partager le portfolio@fr

 

Note :  Lors de l’instanciation du portfolio, le rôle all doit être partagé avec l’utilisateur public.

 

 

Cas généraux. Les deux tableaux ci-dessous expliquent et précisent la valeur des paramètres de la boîte de partage.

Rôle qui partage

Habituellement l’étudiant mais tous les usagers qui ont un rôle défini dans le portfolio peuvent aussi partager.

Rôle du destinataire

Le rôle qu’aura le destinataire dans le portfolio (par exemple « superviseur » – le destinataire pourra faire ce que peut faire un superviseur).

Rôles et/ou courriels 

Le rôle du destinataire dans le portfolio ou encore son courriel :

- Par exemple si « superviseur », le message de partage sera envoyé à la personne qui a le rôle superviseur dans le portfolio. L'usager n'entre pas à la main la personne avec qui partager, ceci est déterminé par le système.

- On peut aussi inscrire directement une adresse courriel si le destinataire est connu.

- Finalement on peut remplacer ce paramètre par « ? », ce qui permet à l’utilisateur qui partage de spécifier le courriel de la personne avec qui partager.

Niveau (0,4)

Le niveau de sécurité détermine si on oblige le destinataire à se connecter à Karuta avec son identifiant et mot de passe avant de voir ce qui lui est partagé.  Il permet aussi à des destinataires qui n’ont pas de compte dans Karuta d’éditer des ressources sans se connecter (niveau 3).

Durée de vie du lien (en heures)

Pour des raisons de sécurité, le designer peut spécifier la durée de vie du lien qui sera envoyé.

button label@en

/ libellé du bouton@fr

Le libellé du menu Partager dans la langue appropriée.  Ne pas oublier le / si bilingue.

Condition

Il est possible de spécifier certaines conditions (facultatif). 

 

Niveau

Conditions

0

Le destinataire existe dans le système Karuta et le portfolio est déjà partagé avec le destinataire dans le bon rôle. Si le destinataire n’est pas connecté au système, il devra le faire en spécifiant son mot de passe.

1

Le destinataire existe dans le système Karuta et le portfolio est déjà partagé avec le destinataire dans le bon rôle. Si le destinataire n’est pas pas connecté au système, il n’aura pas à le faire : en cliquant sur l’hyperlien, il sera automatiquement dirigé vers la bonne section du portfolio.

2

Le destinataire existe dans le système Karuta mais le portfolio n’est pas déjà partagé avec le destinataire dans le bon rôle.  Le système va partager le portfolio dans le bon rôle avec le destinataire. Si le destinataire n’est pas pas connecté au système, il n’aura pas à le faire : en cliquant sur l’hyperlien, il sera automatiquement dirigé vers la bonne section du portfolio.

3

Le destinataire n’existe pas dans le système Karuta. Le système va automatiquement lui créer un compte et lui partager le portfolio dans le bon rôle. Le compte créé est temporaire.

4

Le destinataire qui reçoit le courriel a accès en lecture seulement au contenu de la section du portfolio partagé avec lui. Lors de l’instanciation, le rôle all doit être partagé avec l’utilisateur public.  Le niveau 4 ressemble beaucoup au cas 1 décrit plus haut mais est beaucoup plus flexible; entre autres on peut limiter le nombre d’heures durant lesquelles un accès en lecture sera possible (cas 5).
